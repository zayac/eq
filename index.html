Eq

A stream-based functional programming language

Download this project as a .zip file Download this project as a tar.gz file

Key Concepts

Eq is a stream-based functional language designed for solving numerical tasks. The lanugage key concepts are:

  • Explicit separation of concurrent and data-dependent parts of a program. This allows us to generate an effective parallel code.
  • Introduction of streams. This allows to design programs using stream networks. It provides a full dataflow network for a compiler. Thus the compiler is able to analyse dataflow and to perform useful optimisations.
  • The usage of LaTeX as a front-end language. Our goal is to provide language which uses natural mathematical terms for algorithm description.

Compiler Details

At the moment the Eq compiler is under active development. The compiler's front-end is being maintained at the moment. Currently we use Python as a back-end language, however we aim to support multiple backends and to work out performance issues.

Build Instructions

In order to build the compiler do the following:

 $ cd build/
 $ cmake ..
 $ make

Compiler supports several build options. CMake cache variables can be defined:

  • Select either to build the lexer only -DBUILD_LEXER=[ON|<OFF>]
  • Select compiler's backend -DBACKEND_LANG=[<python>|haskell]
  • Enable or disable functions as a first-class objects -DFIRST_CLASS_FUNCTIONS=[<ON>|OFF]

Compiler Test Execution

In case you want to run a test bundle:

$ ctest

or

$ make test

A verbose output of the build is supported:

make VERBOSE=1

It's possible to execute tests separately:

  • Tests compilation of a correct syntax on different stages (parser, type checking and code generating). ctest -R run_test
  • Tests invalid syntax processing and error handling $ ctest -R fail_test
  • Looks for memory leaks in the compiler ctest -R memory_test

Examples

  • MD5 hash generation can be found in tests/md5/md5.tex file.
